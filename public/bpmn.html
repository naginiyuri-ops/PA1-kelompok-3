Dokumentasi BPMN

Rincian Business Process Model and Notation (BPMN) dalam bentuk tekstual interaktif untuk sistem informasi Geosite Balige, Meat, Batu Basiha, dan Liang Sipege.

1

Proses Login Admin

1

Admin membuka halaman /login pada browser.

2

Admin mengisi form autentikasi (username/email dan password).

3

Sistem menerima request dan memvalidasi kecocokan data dengan database.

Jika gagal: Sistem menampilkan pesan error dan mengembalikan admin ke form login.

Jika berhasil: Sistem membuat session autentikasi yang valid.

4

Admin diarahkan secara otomatis ke halaman Dashboard Admin.

5

Admin menyelesaikan pekerjaan dan mengklik tombol "Logout".

6

Sistem menghancurkan session aktif dan mengarahkan kembali ke halaman Login.

2

Proses Lupa & Reset Password Admin

1

Admin membuka halaman /forgot-password.

2

Admin mengisi form dengan alamat email yang terdaftar.

3

Sistem memverifikasi keberadaan email di dalam database.

Jika tidak ditemukan: Sistem menampilkan pesan error peringatan.

Jika ditemukan: Sistem men-generate token reset password yang unik.

4

Sistem mengirimkan email berisi tautan (link) reset password ke email admin.

5

Admin membuka email, mengklik tautan, dan diarahkan ke halaman /reset-password/{token}.

6

Admin memasukkan password baru dan melakukan konfirmasi password.

7

Sistem memvalidasi kecocokan token dan format password.

Jika token kedaluwarsa/salah: Sistem menolak dan menampilkan pesan error.

Jika valid: Sistem memperbarui data password di database (di-hash ulang).

8

Admin diarahkan kembali ke halaman Login untuk masuk menggunakan password baru.

3

Proses Manajemen (CRUD) Destinasi Wisata

A. Tambah Destinasi
1

Admin membuka menu "Destinasi" (Alam/Buatan/Budaya) lalu mengklik "Tambah".

2

Admin melengkapi form: Judul, Deskripsi, Lokasi, Jam Operasional, Harga, Tags, Gambar, dan Status.

3

Sistem memvalidasi kelengkapan form dan aturan file (maksimal 5MB).

Jika gagal: Proses dibatalkan, pesan error ditampilkan.

Jika berhasil: File gambar diunggah ke storage `public/image/destinations/{kategori}/`.

4

Sistem menyimpan record ke tabel `destinations` dan mengarahkan admin ke daftar destinasi dengan notifikasi sukses.

B. Edit Destinasi
1

Admin mengklik tombol "Edit" pada baris data destinasi terpilih.

2

Sistem memuat data lama dari database ke dalam form.

3

Admin melakukan penyesuaian data (teks atau penggantian gambar) lalu mengklik "Perbarui".

4

Sistem memvalidasi input perubahan.

Jika gambar diubah: Gambar lama dihapus dari storage, gambar baru diunggah.

Jika gambar tidak diubah: Path gambar lama dipertahankan.

5

Sistem melakukan UPDATE pada tabel `destinations` dan kembali ke halaman daftar.

C. Hapus Destinasi
1

Admin mengklik tombol "Hapus" pada data tertentu.

2

Sistem memunculkan popup konfirmasi untuk mencegah penghapusan tidak disengaja.

3

Admin menyetujui konfirmasi penghapusan.

4

Sistem mencari file gambar fisik di storage dan menghapusnya.

5

Sistem menjalankan DELETE pada record di tabel `destinations`.

4

Proses Manajemen Master Data (Berita, UMKM, Penginapan, Biodiversitas, dll)

1

Admin membuka menu modul yang bersangkutan (misal: Berita atau UMKM).

2

Admin melakukan aksi Tambah, Edit, atau Hapus dengan mengisi atribut sesuai tabel.

3

Sistem melakukan fungsi standar (Generate Slug otomatis dari judul, Validasi Input, Manajemen File Upload foto).

4

Admin mengatur "Toggle Status" (Aktif/Nonaktif) pada daftar data.

Jika Aktif: Data akan dirender pada antarmuka publik secara real-time.

Jika Nonaktif: Data disembunyikan dari halaman publik namun aman di database.

5

Proses Interaksi Pengguna (Frontend & Pencarian)

1

Pengguna mengakses Homepage dan bernavigasi ke halaman detail (Destinasi, Berita, Fasilitas).

2

Sistem merender antarmuka dinamis hanya untuk konten dengan status Aktif.

3

Pengguna melihat detail suatu Berita atau Informasi dan mengklik halamannya.

View Counter: Sistem secara latar belakang memanggil API POST untuk mencatat inkrementasi `views + 1`.

4

Pengguna mencari sesuatu via Search Bar Global (menginput kata kunci).

5

Sistem mengirim request AJAX secara real-time (Live Search) ke API backend (/search).

Sistem menyaring basis data pada multi-tabel (Destinasi, Berita, UMKM, Penginapan, Biodiversitas, dll).

6

Sistem memunculkan daftar rekomendasi instan (dropdown suggestion).

Jika hasil di-klik / Enter: Pengguna akan diarahkan ke laman `/search-results?q=keyword` untuk penelusuran lebih mendalam dan spesifik (Paginasi tersedia).

6

Proses Pengaturan Informasi Kontak Web

1

Admin membuka menu "Pengaturan Kontak" dari navigasi sidebar.

2

Admin memasukkan data pembaruan kontak lembaga (Alamat operasional, Nomor Telepon, Email, URL akun sosial media, dan kode embed iFrame Google Maps).

3

Admin menekan tombol simpan perubahan.

4

Sistem memperbarui baris data khusus di tabel kontak (struktur single-record).

Pembaruan langsung terefleksi secara global pada halaman publik /kontak, Footer, dan panel tautan cepat.
